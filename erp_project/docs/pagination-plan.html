Context

Why this change

Every master page today runs an unbounded SELECT * at page load and filters in memory on the client. With a small seed database this is invisible. As vendors, materials, SKUs, and BOMs accumulate, the page load time, server memory, and browser render cost will grow without bound.

Goal: Push filtering, searching, and pagination into the database so the app always fetches only the 20 rows being displayed — regardless of how large the tables grow.
Approach — URL-driven server-side pagination: Next.js App Router server components already receive searchParams. Every master page will accept ?page=1&size=20&search=term&type=rm. No new GET API routes needed. router.refresh() in the Add/CSV dialogs already works — it re-runs the server page with current URL params, keeping the user on the same page after an insert.
Design

Architecture change

Before — fetches everything
MariaDB
↓ SELECT * (all rows)
Server Component
query<T>(sql.selectAll)
↓ initialRows: T[] (entire table)
Client Component
.filter() in memory
As data grows: slow load, wasted memory, browser lag.
After — fetches a slice
URL: ?page=2&size=20&search=acme
↓ searchParams
Server Component
paginate<T>(sql, params)
↓ LIMIT 20 OFFSET 20 + COUNT(*)
MariaDB
↑ rows: T[20] + total: number
Client Component
render rows + <PaginationBar />
Constant load time. DB does the work it is built for.

lib/pagination.ts

Two exports: parsePaginationParams safely reads URL params (clamps page to ≥1, size to 5–100), and paginate<T> runs the data query and COUNT query in parallel and returns a typed PageResult.

lib/pagination.ts New 
import { query } from "@/lib/db"

export type PaginationParams = { page: number; size: number; offset: number }

export function parsePaginationParams(
  sp: Record<string, string | string[] | undefined>,
  defaults = { page: 1, size: 20 }
): PaginationParams {
  const page = Math.max(1, parseInt(String(sp.page ?? defaults.page), 10) || 1)
  const size = Math.min(100, Math.max(5, parseInt(String(sp.size ?? defaults.size), 10) || 20))
  return { page, size, offset: (page - 1) * size }
}

export type PageResult<T> = {
  rows: T[]
  total: number
  page: number
  pageSize: number
}

export async function paginate<T>(
  dataQuery: string,
  dataParams: any[],
  countQuery: string,
  countParams: any[],
  page: number,
  size: number
): Promise<PageResult<T>> {
  const [rows, countRows] = await Promise.all([
    query<T>(dataQuery, dataParams),
    query<{ total: number }>(countQuery, countParams),
  ])
  return { rows, total: Number(countRows[0]?.total ?? 0), page, pageSize: size }
}

"use client"

import { useRouter, usePathname, useSearchParams } from "next/navigation"
import { Button } from "@/components/ui/button"
import { ChevronLeft, ChevronRight } from "lucide-react"

export function PaginationBar({
  total,
  page,
  pageSize,
}: {
  total: number
  page: number
  pageSize: number
}) {
  const router       = useRouter()
  const pathname     = usePathname()
  const searchParams = useSearchParams()

  function navigate(updates: Record<string, string>) {
    const params = new URLSearchParams(searchParams.toString())
    for (const [k, v] of Object.entries(updates)) {
      if (v) params.set(k, v)
      else   params.delete(k)
    }
    router.push(`${pathname}?${params.toString()}`)
  }

  const totalPages = Math.max(1, Math.ceil(total / pageSize))
  const from = total === 0 ? 0 : (page - 1) * pageSize + 1
  const to   = Math.min(page * pageSize, total)

  return (
    <div className="flex items-center justify-between px-4 py-3 border-t border-border">
      <div className="flex items-center gap-2 text-sm text-muted-foreground">
        <span className="text-xs">Rows per page</span>
        <select
          value={pageSize}
          onChange={(e) => navigate({ size: e.target.value, page: "1" })}
          className="h-7 rounded border border-input bg-background px-2 text-xs"
        >
          {[10, 20, 50, 100].map((n) => (
            <option key={n} value={n}>{n}</option>
          ))}
        </select>
      </div>
      <div className="flex items-center gap-3 text-sm text-muted-foreground">
        <span className="text-xs">
          {total === 0 ? "No records" : `${from}–${to} of ${total}`}
        </span>
        <div className="flex gap-1">
          <Button
            variant="outline"
            size="icon-sm"
            disabled={page <= 1}
            onClick={() => navigate({ page: String(page - 1) })}
            aria-label="Previous page"
          >
            <ChevronLeft />
          </Button>
          <Button
            variant="outline"
            size="icon-sm"
            disabled={page >= totalPages}
            onClick={() => navigate({ page: String(page + 1) })}
            aria-label="Next page"
          >
            <ChevronRight />
          </Button>
        </div>
      </div>
    </div>
  )
}

components/masters/UrlSearchInput.tsx

Wraps the existing SearchInput with a 350 ms debounce that pushes ?search=term&page=1 to the URL. Keeps a local useState so keystrokes feel instant. Reads initialValue from the server so a page reload never shows an empty box while rows are filtered below.

components/masters/UrlSearchInput.tsx New 
"use client"

import { useState, useEffect, useRef } from "react"
import { useRouter, usePathname, useSearchParams } from "next/navigation"
import { SearchInput } from "@/components/masters/SearchInput"

export function UrlSearchInput({
  initialValue = "",
  placeholder,
  debounceMs = 350,
}: {
  initialValue?: string
  placeholder?: string
  debounceMs?: number
}) {
  const [value, setValue]    = useState(initialValue)
  const router               = useRouter()
  const pathname             = usePathname()
  const searchParams         = useSearchParams()
  const timerRef             = useRef<ReturnType<typeof setTimeout>>(undefined)

  // Sync when server re-renders with a new initialValue (e.g. after clear filters)
  useEffect(() => { setValue(initialValue) }, [initialValue])

  function handleChange(v: string) {
    setValue(v)
    clearTimeout(timerRef.current)
    timerRef.current = setTimeout(() => {
      const params = new URLSearchParams(searchParams.toString())
      if (v) params.set("search", v)
      else   params.delete("search")
      params.set("page", "1")
      router.push(`${pathname}?${params.toString()}`)
    }, debounceMs)
  }

  return (
    <SearchInput
      value={value}
      onChange={handleChange}
      placeholder={placeholder}
    />
  )
}

Query file additions

Add selectPaginated and countAll alongside the existing selectAll in each query file. Never remove selectAll — it is still used by the Add/CSV wizard dropdowns which need the full list.

SQL pattern: ? IS NULL OR col LIKE ? — when no search is active pass null as the first param and MySQL short-circuits without scanning rows. Param order for data query: [like, like, like, typeParam, typeParam, size, offset]. Drop the last two for the count query.

Pattern — Vendors example

Server page.tsx (full replacement)

Auth and permission guards are unchanged. The only additions are reading searchParams, calling parsePaginationParams, and using paginate instead of the bare query.

app/masters/vendors/page.tsx Update 
import { auth } from "@/lib/auth"
import { resolveAccess } from "@/lib/permissions"
import { redirect } from "next/navigation"
import { parsePaginationParams, paginate } from "@/lib/pagination"
import { vendors } from "@/lib/queries/vendors"
import type { Vendor } from "@/types/masters"
import VendorsClient from "./VendorsClient"

export default async function VendorsPage({
  searchParams,
}: {
  searchParams: Promise<Record<string, string | string[] | undefined>>
}) {
  const session = await auth()
  if (!session) redirect("/auth/signin")
  const userId = parseInt(session.user.id)
  const access = await resolveAccess(userId, session.user.roles, "/masters")
  if (access === "none") redirect("/auth/unauthorized")

  const sp         = await searchParams
  const { page, size, offset } = parsePaginationParams(sp)
  const search     = String(sp.search ?? "")
  const typeFilter = String(sp.type   ?? "")
  const like       = search     ? `%${search}%` : null
  const type       = typeFilter ? typeFilter     : null

  const { rows, total } = await paginate<Vendor>(
    vendors.selectPaginated,
    [like, like, like, type, type, size, offset],
    vendors.countAll,
    [like, like, like, type, type],
    page,
    size
  )

  return (
    <div className="p-6">
      <div className="mb-5">
        <h1 className="text-2xl font-bold tracking-tight">Vendors</h1>
        <p className="text-muted-foreground text-sm mt-1">All registered vendors</p>
      </div>
      <VendorsClient
        rows={rows}
        total={total}
        page={page}
        pageSize={size}
        currentSearch={search}
        currentType={typeFilter}
      />
    </div>
  )
}

Verification checklist

Run after wiring each master page. Final npm run build at the end.

  • Navigate to page — only 20 rows load by default (check Network tab: small JSON payload)
  • Type in search box, wait 350 ms — rows filter via DB, URL updates to ?search=term&page=1
  • Click Next / Prev — URL page param increments/decrements, correct slice shown
  • Change page size to 50 — refetches 50 rows, URL becomes ?size=50&page=1
  • Add a record via the dialog — router.refresh() stays on same page and keeps filters
  • Import CSV — same, user stays on current page with current filters
  • Click "Clear filters" — search and type params removed, back to page 1
  • (Packing / Raw Materials) Toggle view — lands on page 1 of the new view
  • Copy URL, open in new tab — same page, search, and filters load correctly (URL is shareable)
  • npm run build — no TypeScript errors, clean output
  • npm run lint — no lint errors
